## Architecture (verified)
Thin server pages under `src/app/admin/**` each call `requirePermission(...)` then render one client component from `src/components/admin/**`. Nav + edge gate + page guards share one map: `ADMIN_PATH_PERMISSIONS` in `src/lib/auth/roles.ts:65`. All data flows through `adminApi` (`src/lib/api/resources/admin.ts`) to `src/app/api/admin/**`. React Query throughout; toasts via `useToast`; shared styles in `src/components/admin/adminUi.module.css`. Quality baseline is already high: consistent loading (`TableSkeleton`), error (`EmptyState tone="error"` + retry), empty states, RTL, `aria-label`s, keyboard row toggles, confirm guards, audit trail with before/after.

## 2. Per-route completeness & UX

- **Pricing** вЂ” Best component in the panel: O(1) dirty Sets (`:83`), Enter-to-next-cell (`:194`), invalid-number feedback (`:202`), filter-change guard against losing edits (`:94`). **Gaps:** no bulk paste/CSV import (API `pricing/route.ts:25` already takes a JSON array вЂ” trivial to wire), no arrow-key vertical nav, no per-row price history sparkline/drilldown, no "hide/unhide price" or "mark stale" action, no copy-yesterday.
- **BilletCard** вЂ” Only `billet` is editable, but `MarketKey = usd|eur|gold18|ounce|billet` (`domain.ts:9`). **Gap:** the other 4 market tickers have no admin entry/override UI at all.
- **Leads/CRM** вЂ” Strong flow. **Gaps:** no pagination/infinite scroll (uses `data.total` but loads all), no assignee/owner, no date-range filter, no CSV export, `LeadDetail` proforma has no line-item editing/discount before issue, no way to add items to a lead, no bulk status change.
- **Requests/Messages** вЂ” Functional. **Gaps:** `RequestsTab` has no filter/pagination; `MessagesTab` has no reply-in-place, no filter, no pagination, only newв†’handled (no re-open).
- **Orders** вЂ” Card + stepper is clean. **Gaps:** strictly linear forward (no back/cancel/hold, no tracking number/carrier, no shipment date entry), no search by ref, no pagination.
- **Warehouse** вЂ” Solid receive form with client validation (`:61`). **Gaps:** no edit of quantity/fee after receipt, no per-customer filter/search, no settlement/billing view, no history.
- **Content** вЂ” Editor is Markdown-only textarea (`ContentQueue.tsx:193`) with a hand-rolled 3-rule preview (`:230`). **Gaps (large):** schema has `coverUrl`, `seo` (jsonb `SeoMeta`), `authorId` (`content.ts:21-33`) вЂ” **none are editable in the UI**. No image upload, no rich editor, no SEO fields, no cover, no tags, no unpublish/revert, no delete, no AI-source diff/provenance view.
- **Catalog** вЂ” SKU CRUD good. **Gaps:** categories & subcategories have full APIs (`catalog/categories`, `catalog/subcategories` + `[id]`) but **no UI** вЂ” you cannot create/rename a category or sub in the panel. `skus.imageUrl` exists (`catalog.ts:29`) but is not editable. No search within category, no bulk activate/deactivate, must select a category before seeing anything.
- **Users** вЂ” Role/active inline selects with last-admin guard server-side and confirm dialogs. **Gaps:** no pagination (loads all), no create-staff/invite, no per-user detail (their leads/orders/AI usage), no session-revoke button, club tier duplicated as a separate table.
- **Settings** вЂ” Excellent: per-key validation, error inline, `isError` guard prevents saving fabricated defaults (`SettingsForm.tsx:49`). **Gaps:** logistics cities and holidays are freeform textareas (no structured rows), no AI-advisor config surface (model/temp/system-prompt caps live in code).
- **Audit** вЂ” Good cursor pagination. **Gaps:** no actor-name resolution (shows raw `actorId`, `:83`), no date-range/action/actor filter, no full-text search, no export, raw JSON only.
- **Dashboard** вЂ” Good tiles. **Gaps:** no charts/trends, AI tile links to `/admin/leads` (wrong вЂ” there's no AI review page to link to).

## 3. Gaps an admin CANNOT do today (highest-impact)

1. **No AI-answer review** вЂ” `aiConversations`/`aiMessages`/`aiUsage` are persisted (`system.ts:49-92`, violations tracked) but there is **no `/admin/ai` route** to read conversations, inspect violations, or see cost trends. This is the flagship product and it's a black box in the admin.
2. **No category/subcategory management UI** вЂ” APIs exist, UI doesn't.
3. **No image upload anywhere** вЂ” no `/api/*/upload`, no media library; `coverUrl`/`imageUrl` are dead fields in the UI.
4. **No article SEO / cover / tags / delete / unpublish** вЂ” publishing pipeline is one-way.
5. **No bulk price import (paste/CSV)** despite an array-accepting API.
6. **No non-billet market tickers** editable.
7. **No pagination** on leads/users/orders/messages/requests (loads full tables).
8. **No global search / command palette**, no cross-entity nav.
9. **Audit actor is an opaque id**; no filtering.
10. **No alerts (Щ‚ЫЊЩ…ШЄвЂЊШіЩ†Ш¬) admin view** despite `alerts` table (`engagement.ts:36`).

### Phase B вЂ” Core feature completion

**B1. Category & subcategory CRUD.** New `catalog/CategoryManager.tsx` (add/rename/reorder/deactivate) wired to existing `catalog/categories` + `catalog/subcategories` APIs. Mount as a tab in `CatalogManager.tsx` alongside SKUs. Add search-within-category box.

**B2. Image upload + media.** New `POST /api/admin/upload` (validate mime/size, store to disk/`public/uploads` or object storage, return URL) + `requirePermission('content:write'|'catalog:write')`. New `components/admin/shared/ImageUpload.tsx` (drag/drop, preview, progress). Wire into A5 SKU form and B3 article editor.

**B3. Article editor upgrade.** In `ContentQueue.tsx` `ArticleEditor`: add cover image (`ImageUpload`), SEO card (title/description/ogImage в†’ `seo` jsonb), tags, author display, and **unpublish/delete** actions (extend `articles/[id]/route.ts`). Replace the 3-rule preview (`:230`) with a proper markdown renderer (reuse the public blog renderer for parity) and add a toolbar (bold/heading/list/link) over the textarea вЂ” keep Markdown as source of truth.
